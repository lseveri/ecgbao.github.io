Pushing to Git returning Error Code 403 fatal: HTTP request failed 解决方案

By 跳跳魔王

发表于 2017-02-08

文章目录

原因可能是使用了 https 协议登录了, 首先你要向 git 的Remote URI 设置自己的身份验证的凭证:

git remote set-url origin https://username@github.com/user/xxx.git

当 git push 的时候,会要求你输入git的密码。

实际上, 如果是 http 鉴权的格式。你也可以设置一个密码:

https://youruser:password@github.com/user/repo.git

这样做你得注意, 你的 github 会存在 .git 目录下面的文本里, 这显然是不可取的做法。
